# 主项目规划：AI Agent 平台

## 一、项目定位

这个项目不是一个简单的聊天页面，而是一个面向团队或业务侧的 `AI Agent Platform`。它的核心是让用户可以创建、配置、测试、发布和管理多个 `Agent`，并持续观察它们的效果、成本和稳定性。

你可以把它理解成一个“智能体运营平台”，目标用户包括：

- 产品经理
- 运营人员
- 内部工具开发者
- 业务方使用者

这个项目更偏“平台化、管理化、产品化”。

---

## 二、为什么这个项目适合冲字节前端

它能同时覆盖下面这些能力：

- `React` 中后台工程能力
- 复杂表单和配置驱动能力
- 流式输出和多面板联动
- 权限、版本、发布、日志等后台能力
- AI 业务理解：模型、Prompt、工具、知识库、记忆、评测
- 工程化和稳定性意识

如果你想把项目做得更贴近真实公司场景，而不是偏“炫技 Demo”，这个项目非常合适。

---

## 三、核心目标

你要做出的不是“一个 Agent”，而是一个“管理多个 Agent 的系统”。这个系统至少要支持：

1. Agent 列表与生命周期管理
2. Agent 配置与调试
3. 会话与运行记录查看
4. Prompt、模型、工具、知识库的组合配置
5. 发布、版本、权限、审计
6. 评测与反馈闭环

---

## 四、项目必须覆盖的核心模块

## 1. Agent 列表与管理

至少包含：

- Agent 列表页
- 创建 Agent
- 编辑 Agent
- 复制 Agent
- 删除 Agent
- 启用/停用 Agent
- 标签、搜索、筛选

你要能讲清：

- Agent 的数据结构如何设计
- Agent 状态如何划分，例如草稿、已发布、停用
- 列表页如何做成可扩展结构

## 2. Agent 配置中心

至少包含：

- 基础信息配置
- System Prompt 配置
- 模型选择与参数配置
- 工具绑定
- 知识库绑定
- Memory 配置
- 安全与输出限制配置

这里是项目的第一大亮点，因为它能体现：

- 配置驱动表单设计
- 复杂状态管理
- 多模块联动
- 可扩展的数据协议

## 3. Prompt 与调试工作台

至少包含：

- 输入测试区
- 流式输出展示
- 中间步骤展示
- token 或耗时信息展示
- 错误提示
- 调试历史记录

这个模块的价值在于：

- 能体现 AI 产品理解
- 能体现流式渲染和异步状态处理能力
- 能让面试官看到你做的不只是后台配置页

## 4. 工具与知识库管理

至少包含两个子模块：

- 工具管理：新增工具、配置工具参数、测试工具可用性
- 知识库管理：查看已绑定知识库、配置召回参数、测试检索结果

你要能讲清：

- 工具和 Agent 的关系是什么
- 知识库检索结果如何影响最终回答
- 为什么工具和知识库要抽象成独立资源

## 5. 会话与运行记录

至少包含：

- 会话列表
- 会话详情
- 单次运行轨迹
- 输入输出记录
- 失败原因展示
- 时间、用户、Agent 等维度筛选

这是很关键的后台能力，因为它能体现：

- 可观测性
- 数据结构设计
- 日志类页面的交互复杂度

## 6. 发布与版本管理

至少包含：

- 版本列表
- 草稿保存
- 发布
- 回滚
- 版本说明

你要能讲清：

- 为什么 Agent 平台必须有版本概念
- 配置变更为什么不能直接覆盖线上版本
- 发布流程如何设计才更安全

## 7. 评测与反馈系统

至少包含：

- 用户反馈收集
- 手动评分
- 结果对比
- 关键指标展示，例如成功率、平均耗时、异常率

这个模块是项目的重要加分点，因为它会让项目更像真正的 AI 平台，而不是普通后台。

## 8. 权限与审计

至少包含：

- 登录与鉴权
- 角色权限
- 页面级权限
- 按钮级权限
- 操作日志

这部分补齐字节会看的后台工程能力。

---

## 五、建议技术栈

- 框架：`React` + `TypeScript`
- 构建：`Vite`
- 路由：`React Router`
- 状态管理：`Zustand` 或 `Redux Toolkit`
- UI：`Ant Design`
- 请求层：`Axios`
- 图表：`ECharts`
- 富文本或 Prompt 编辑：`Monaco Editor` 或 `CodeMirror`
- 测试：`Vitest` + `React Testing Library`
- 规范：`ESLint` + `Prettier` + `Husky` + `lint-staged`

如果你想进一步加分，可以再补：

- 国际化
- 深色模式
- 埋点与错误监控
- 版本对比视图

## 六、你必须做出的技术亮点

至少完成下面 `6` 类中的 `4` 类。

## 1. 配置驱动 Agent Schema

你要设计一套 Agent 配置协议，至少包括：

- 基础信息
- Prompt 配置
- 模型配置
- 工具配置
- 知识库配置
- Memory 配置

亮点在于：

- 表单不是写死的
- 方便后续扩展新的 Agent 能力
- 可以体现你的抽象能力

## 2. 流式调试面板

你要完成：

- 流式输出
- 运行状态切换
- 加载中与失败态
- 中间步骤展示

亮点在于：

- 前端异步状态管理更复杂
- 能体现 AI 场景的实时交互能力

## 3. 多面板联动

建议做成这种结构：

- 左侧 Agent 列表
- 中间配置或调试面板
- 右侧详情、日志、评测面板

亮点在于：

- 能体现中后台复杂布局能力
- 状态同步和组件通信难度更高

## 4. 版本和发布系统

你要完成：

- 草稿保存
- 发布
- 历史版本
- 版本说明
- 回滚

亮点在于：

- 具备平台产品思维
- 容易在面试里体现系统设计感

## 5. 评测与反馈闭环

你要完成：

- 会话评价
- 结果打分
- 指标看板
- 失败案例查看

亮点在于：

- 能体现你不仅关注“能运行”，还关注“效果如何”

## 6. 工程化与稳定性

至少完成：

- 统一请求封装
- 统一异常处理
- 页面级空态/异常态
- 测试覆盖核心模块
- 权限控制
- 操作日志

## 八、项目结构建议

- `src/app`：应用入口、路由、Provider
- `src/layouts`：平台布局
- `src/pages`：页面级入口
- `src/features/agent`：Agent 相关业务域
- `src/features/session`：会话与调试
- `src/features/evaluation`：评测中心
- `src/features/system`：权限、日志、设置
- `src/components`：通用组件
- `src/services`：请求层
- `src/store`：全局状态
- `src/types`：类型和协议
- `src/utils`：工具函数
