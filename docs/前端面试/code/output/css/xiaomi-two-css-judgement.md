```html
<style>
.a #b { color: blue; }
.b { color: red; }
</style>
<body>
  <div class="a" id="b">
    <div class="b" id="a">
      text
    </div>
  </div>
</body>
```

问题：`text` 最终是什么颜色？

`text` 是红色。

### 分析

- `.b { color: red; }` 直接命中了内层这个 `div`，也就是直接包裹文本 `text` 的元素。
- `.a #b { color: blue; }` 的意思是：类名为 `a` 的祖先元素内部，存在一个后代元素 `id="b"`。
- 但当前 DOM 结构里，`class="a"` 和 `id="b"` 在同一个元素上，不是祖先与后代关系。
- 所以 `.a #b` 这个后代选择器并没有命中任何元素。
- 既然蓝色规则没有生效，而内层 `.b` 命中了文本最近的元素，最终文本颜色就是红色。

### 易错点

- `.a #b` 和 `.a#b` 不是一回事。
- `.a #b` 表示后代选择器，中间有空格，要求是两个层级。
- `.a#b` 表示同一个元素同时满足 `class="a"` 和 `id="b"`。

### 延伸理解

- 即使外层元素命中了颜色规则，文本也会优先使用离自己最近、直接命中的元素颜色。
- `color` 本身是可继承属性，但“元素自身命中的样式”优先级高于“从父元素继承来的样式”。
