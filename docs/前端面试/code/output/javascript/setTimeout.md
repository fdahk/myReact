## 1. `var + setTimeout` 输出题

### 题目一

```js
for (var i = 0; i < 5; i++) {
  setTimeout(() => {
    console.log(i);
  }, 100);
}
```

### 分析

- `var` 是函数作用域，不会为每次循环单独创建一个新的块级绑定。
- 这 5 个定时器回调拿到的是同一个外层变量 `i`。
- 当定时器真正执行时，`for` 循环早已结束，此时 `i === 5`。
- 所以 5 次输出读到的都是同一个最终值 `5`。

### 题目二

```js
for (var i = 0; i < 5; i++) {
  function printText(text) {
    setTimeout(() => {
      console.log(text);
    }, 100);
  }
  printText(i);
}
```

### 分析

- 虽然外层仍然使用了 `var`，但 `printText(i)` 在每次循环时都把当前的 `i` 作为参数传了进去。
- 参数 `text` 是 `printText` 这次调用自己的局部变量。
- 每次调用都会形成各自独立的闭包环境，因此 5 个定时器分别保存了不同的 `text` 值。
- 所以最终会按顺序输出 `0 1 2 3 4`。

### 题目三

```js
for (var i = 0; i < 5; i++) {
  setTimeout(() => {
    var temp = i;
    console.log(temp);
  }, 100);
}
```

### 分析

- 很多人以为 `var temp = i` 能把每轮循环的值“缓存下来”，但这里的赋值发生在定时器回调执行时，不是循环执行时。
- 等到回调执行，外层 `i` 已经变成 `5`。
- 所以每次进入回调时，`temp` 都会被赋值成 `5`。
- 这段代码只是把最终的 `i` 再拷贝给了 `temp`，并没有形成每轮独立快照。
