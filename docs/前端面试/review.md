如果你的目标是进入字节跳动做前端或后端，核心不是“会不会某个框架”，而是你是否具备下面这几类能力：

## 一、通用条件

### 1. 计算机基础要扎实
你至少需要具备这些能力：

- 数据结构与算法：数组、链表、栈、队列、树、图、哈希、排序、二分、DFS/BFS、动态规划
- 计算机网络：HTTP/HTTPS、TCP/UDP、DNS、CDN、缓存、Cookie/Session、跨域
- 操作系统：进程/线程、并发、锁、内存、IO
- 数据库：MySQL 索引、事务、隔离级别、锁、SQL 优化；Redis 常见数据结构与使用场景
- 工程能力：Git、CI/CD、日志、监控、测试、排查问题

### 2. 工程能力不能只停留在“会写功能”
字节比较看重你是否能写出“可维护、可扩展、可定位问题”的代码：

- 有清晰的模块设计能力
- 能写测试，不只是写业务代码
- 能做性能优化，而不是只会堆功能
- 能定位线上问题，并给出解决方案
- 能写文档、做复盘、讲清楚设计取舍

### 3. 业务理解和成长性
大厂不只看技术，还看你是否有“工程师思维”：

- 能理解需求本质，而不是机械开发
- 能关注用户体验、稳定性、效率、成本
- 能主动发现问题并推动改进
- 沟通清晰，协作顺畅，反馈及时

# 前端工程师需要额外满足的条件

## 前端核心能力

- HTML / CSS / JavaScript / TypeScript 扎实
- 熟悉浏览器原理：渲染流程、重排重绘、事件循环、缓存机制
- 熟练掌握 React 或 Vue，最好对源码机制有一定理解
- 会做组件化、状态管理、路由设计、权限设计
- 有性能优化经验：首屏、懒加载、打包体积、缓存、长列表、图片优化
- 有工程化能力：Webpack/Vite、Lint、测试、发布流程、监控埋点
- 有跨端或全栈协作能力更加分：Node.js、SSR、Hybrid、小程序、移动端适配

## 字节前端更看重什么

- JavaScript 基础非常硬
- React 生态熟练
- 工程化和性能优化有真实经验
- 能把复杂交互做稳定
- 对“用户体验”和“数据驱动优化”有意识

---

# 后端工程师需要额外满足的条件

## 后端核心能力

- 至少精通一门后端语言：Go、Java、Python、C++ 中的一种
- 熟悉 Web 服务开发：鉴权、参数校验、异常处理、日志、监控
- 熟悉数据库设计和 SQL 优化
- 熟悉 Redis、MQ、缓存、一致性、幂等
- 熟悉高并发、高可用、分布式基础概念
- 能做接口设计、服务拆分、性能调优、故障排查
- 能从业务角度设计数据模型和服务边界

## 字节后端更看重什么

- 基础是否扎实，尤其是网络、并发、数据库
- 是否有真实项目复杂度，而不只是 CRUD
- 是否理解服务稳定性和性能瓶颈
- 是否能做系统设计，而不只是写接口
- 是否能独立排查线上问题

# 二、你该有什么量化指标来证明自己

重点不是“包装”，而是让别人一眼看出你具备交付能力。下面这些指标比较有说服力。

## 1. 算法与基础能力指标

### 建议标准
- LeetCode 总题量：`200+`
- 中等题：`120+`
- 困难题：`20+`
- 热题 Top 100：`80% 以上能独立完成`
- 常见题型二刷率：`70%+`
- 模拟面试时：`30~45 分钟内完成一道中等题，并能讲清复杂度`

### 更有竞争力的标准
- LeetCode：`300~500`
- 能稳定输出题解思路，而不是背答案
- 八股不是死记，而是能结合项目解释

---

## 2. 项目经历指标

你至少应该有 `2~3 个能深挖的项目`，每个项目都能回答：

- 为什么这么设计
- 遇到了什么问题
- 如何权衡方案
- 性能怎么优化
- 稳定性怎么保证
- 最终效果如何量化

### 前端项目建议量化指标
- 首屏加载时间下降：`30%+`
- 打包体积下降：`20%+`
- 页面渲染耗时下降：`20%+`
- 线上报错率下降：`30%+`
- 组件复用率提升：`40%+`
- 页面性能评分达到：`Lighthouse 90+`

### 后端项目建议量化指标
- 接口平均响应时间下降：`30%+`
- P95 / P99 延迟明显改善
- QPS 提升：`2x` 或以上
- 数据库慢查询下降：`50%+`
- 缓存命中率达到：`80%+`
- 线上故障恢复时间缩短：`50%+`

---

## 3. 实习/真实业务指标

如果你有实习，这部分价值很高。

### 有说服力的证明方式
- 独立负责过 `1 个完整模块`
- 真正上线过功能，并有用户或业务数据反馈
- 参与过线上问题排查
- 做过需求评审、接口评审、技术方案评审
- 有明确业务结果，例如：
  - 用户转化率提升 `x%`
  - 页面停留时长提升 `x%`
  - 工单量下降 `x%`
  - 人工处理时间下降 `x%`

### 建议你具备

- 有完整日志、监控、告警意识
- 能写规范文档、技术方案、复盘文档
- 至少做过一次完整性能优化或稳定性治理

- 错误告警平均发现时间缩短 `x%`

# 字节计划

## 适用前提

这份计划默认你当前目标是：

- 岗位方向：前端工程师
- 当前状态：基础较强，但需要把项目证明、面试表达、算法稳定性和求职材料进一步打磨到能冲字节的水平

---

## 一、最终目标

你要达成的不是“继续学前端”，而是进入一种可稳定投递字节前端岗位的状态。对应的结果应该是：

- 有 `1 个` 能被深挖 `15 分钟+` 的高质量主项目
- 有 `1 个` 补充型项目，用来证明工程化、组件化、性能优化或平台能力
- 算法达到 `150~220` 题，且最近 `30` 题正确率稳定在 `80%+`
- 前端高频知识题库达到 `180~250` 个，并能结构化表达
- 完成 `8 次+` 模拟面试
- 有成型的简历、GitHub、项目文档、技术文章作为外部证明

## 二、字节前端达标条件

### 1. 基础必须过硬

你至少要能稳定回答下面这些模块：

- JavaScript：原型链、闭包、`this`、事件循环、Promise、异步编程、模块化
- 浏览器：渲染流程、重排重绘、缓存、跨域、安全、从输入 URL 到页面展示
- 网络：HTTP/HTTPS、状态码、缓存协商、TCP/UDP、CDN、WebSocket
- React：Hooks、状态管理、渲染机制、性能优化、组件抽象、常见陷阱
- 工程化：Webpack/Vite、Babel、构建优化、代码规范、测试、发布流程

### 2. 项目必须能证明你真的做过

你至少需要 `2 个` 能说服面试官的项目：

- `1 个` 主项目：复杂度高、用户链路完整、包含真实工程问题
- `1 个` 加分项目：例如组件库、低代码编辑器、性能监控平台、SSR 项目、可视化系统

每个项目都必须能讲清：

- 为什么这样设计
- 难点在哪里
- 怎么排查问题
- 做过哪些优化
- 指标怎么量化
- 如果重做会怎么改

### 3. 算法不能只是刷过

你要做到：

- 中等题能在 `20~30` 分钟内完成
- 能口头说明思路、复杂度、边界条件
- 高频题型至少系统覆盖一次并完成二刷

### 4. 面试表达要有结构

你要准备好：

- `2~3` 分钟自我介绍
- 项目 `1` 分钟版、`3` 分钟版、深挖版
- 高频题回答模板：结论、原理、场景、追问

## 三、量化指标

### 1. 基础题库指标

- JavaScript、浏览器、网络、React、工程化五大模块，各整理 `30~50` 个高频问题
- 累计沉淀 `180~250` 个问答卡片
- 任意抽一个高频题，`2` 分钟内能给出结构化回答

### 2. 算法指标

- 总题量：`150~220`
- 核心题型覆盖：双指针、二分、链表、栈、队列、树、DFS/BFS、回溯、动态规划、贪心、哈希
- 最近 `30` 题正确率：`80%+`
- 每周至少 `1` 次限时模拟手写

### 3. 项目指标

- 主项目有效业务代码量建议达到 `5k~15k`
- 主项目至少包含 `3` 个可深挖技术难点
- 至少做出 `2` 项可量化优化，例如：
  - 首屏时间下降 `30%+`
  - 打包体积下降 `20%+`
  - 复杂页面渲染耗时下降 `40%+`

### 4. 输出证明指标

- GitHub 至少有 `2` 个高质量仓库
- 每周至少 `4` 天有效提交
- 输出 `4~8` 篇技术文章或项目复盘
- 每个简历项目都能对应仓库、文档或演示说明

### 5. 面试指标

- 模拟面试累计 `8` 次以上
- 每次模拟后都要输出复盘
- 主项目介绍可稳定支撑 `15` 分钟深挖

## 四、12 周执行计划

### 第 1 阶段：定位与材料搭建（第 1~2 周）

目标：先明确主项目和证明材料，不再随机学习。

本阶段任务：

- 确定 `1` 个主项目方向和技术栈
- 梳理现有经历，筛出最适合写进简历的 `2~3` 个项目
- 建立六大模块知识库：JavaScript、浏览器、网络、React、工程化、算法
- 整理第一批高频题答案
- 产出简历 `V1`

阶段验收：

- 主项目题目已确定
- 简历 `V1` 完成
- 高频题卡片达到 `40+`
- 算法累计 `20~30` 题

### 第 2 阶段：主项目深挖与基础强化（第 3~6 周）

目标：做出真正能讲的主项目，并系统补强基础表达。

本阶段任务：

- 每周推进主项目 `1` 个核心模块
- 每周完成 `1` 个难点功能并输出复盘
- 每天按主题整理高频题，形成“原理 + 场景 + 追问”的答案结构
- 保持算法日更
- 开始写技术文章

推荐主项目方向：

- 企业级后台管理系统
- 可视化低代码页面编辑器
- 性能监控/日志分析平台前端
- 组件库 + 文档站

阶段验收：

- 主项目完成度达到 `60%+`
- 形成 `2` 个以上可讲技术难点
- 高频题卡片累计 `100+`
- 算法累计 `80~100` 题
- 技术文章累计 `2` 篇以上

### 第 3 阶段：性能、工程化、深度表达（第 7~9 周）

目标：把项目从“能用”打磨到“能面”。

本阶段任务：

- 为主项目补性能优化和工程化建设
- 补测试、Lint、错误处理、模块拆分、文档
- 做一轮数据化优化前后对比
- 开始项目深挖问答训练
- 更新简历 `V2`

本阶段必须补齐：

- 至少 `2` 项性能优化，并提供数据对比
- 至少 `1` 张架构图或模块拆解图
- 至少 `1` 份项目难点复盘文档
- 至少 `1` 篇项目优化总结文章

阶段验收：

- 主项目完成并可演示
- 有 `2~3` 个量化优化指标
- 高频题卡片累计 `160+`
- 算法累计 `130~160` 题
- 模拟面试累计 `3` 次以上

### 第 4 阶段：面试冲刺与投递（第 10~12 周）

目标：把准备结果转化成面试反馈和 offer 概率。

本阶段任务：

- 每周进行 `2~3` 次模拟面试
- 每次模拟后补短板题单
- 按岗位描述定向修改简历
- 与字节同级别公司并行投递，增加面试反馈样本
- 每场正式面试后做复盘，更新题库和表达模板

阶段验收：

- 模拟面试累计 `8+`
- 算法累计 `150~220` 题
- 高频题卡片累计接近 `200`
- 简历 `V3` 完成
- 完成第一轮正式投递与复盘
