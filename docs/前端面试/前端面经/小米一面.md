# 小米一面

## 基础概念

### 块级作用域、函数作用域、全局作用域

#### 1. 全局作用域

- 在最外层定义的变量，整个程序都可以访问。
- 浏览器里用 `var` 定义的全局变量，通常会挂到 `window` 上；`let` / `const` 不会直接挂到 `window`。
- 全局变量生命周期长，容易造成命名污染，一般应尽量减少。

#### 2. 函数作用域

- `var` 和函数声明在函数内部定义时，作用域被限制在当前函数内。
- 函数执行时会创建自己的执行上下文，内部可以访问外部变量，外部不能直接访问内部变量。
- 闭包的本质，就是内部函数引用了外部函数作用域中的变量。

#### 3. 块级作用域

- `let` / `const` 会形成块级作用域，常见于 `if`、`for`、`while`、`switch`、单独的大括号块。
- `for (let i = 0; i < 5; i++)` 里的 `i` 也属于块级作用域，不是必须手写额外的 `{}` 才算。
- 更准确地说，`for` 语句本身会为 `let` / `const` 创建一个循环作用域；如果循环体里再写 `{}`，那只是循环体自己的代码块。
- 在 `for (let i...)` 中，`i` 的可见范围只在当前 `for` 语句内部，循环外不能访问。
- 对于异步场景，`for (let i...)` 还有一个关键特性：每次迭代都会创建一个新的 `i` 绑定，所以定时器、事件回调拿到的是各轮独立的值。
- 但 `while` 和 `if` 要特别区分：`while (i) let x;`、`if (flag) let x = 1;` 这种“没写 `{}` 就直接写 `let` 声明”的形式在 JavaScript 中是非法语法，不存在“它到底是全局还是块级”的问题，因为代码根本过不了解析。
- 原因是 `let` / `const` 属于词法声明，不能出现在这类语句的单条语句位置里，必须放进一个块里。
if、while、else 这类语句在“不写 {}”时，后面只能接“一条可执行语句（statement）”，但 let / const 这种声明不允许直接放在这个位置。
- 所以可以这样记：
  - `for (let i...)` 比较特殊，`for` 头部本身就允许词法声明。
  - `while` / `if` / `else` 如果不写 `{}`，后面只能跟一条普通语句，不能直接跟 `let` / `const` 声明。

- 块级作用域能避免变量泄漏，尤其适合循环和条件分支。
- `let` / `const` 存在暂时性死区（TDZ），即在声明前不能访问。

#### 4. 三者的核心区别

- 可见范围不同：全局最大，函数次之，块级最小。
- 变量提升行为不同：`var` 会提升并初始化为 `undefined`，`let` / `const` 也会提升但不能在声明前使用。
- 工程实践中优先使用 `const`，其次 `let`，尽量避免 `var`。

### ES6 特性

可以按以下几个维度回答：

#### 1. 变量声明

- `let`、`const` 引入块级作用域。
- 解决了 `var` 的变量提升、重复声明、循环闭包等问题。

#### 2. 函数能力增强

- 箭头函数：没有自己的 `this`，`this` 取决于外层词法环境。
- 默认参数。
- 剩余参数 `...args`。
- 展开运算符 `...`。

#### 3. 字符串和模板

- 模板字符串支持插值和换行，适合拼接 HTML、SQL、文案等。

深入一点可以这样讲：

#### 1. 模板字符串的核心能力

- 使用反引号 `` ` ` `` 定义字符串。
- 支持 `${}` 插值，可以直接嵌入表达式。
- 支持天然换行，不需要手动写 `\n`。

例如：

```js
const name = "Tom";
const age = 20;

const text = `name: ${name}, next year: ${age + 1}`;
console.log(text); // name: Tom, next year: 21
```

#### 2. 为什么比字符串拼接更好

- 可读性更强，尤其是多变量拼接时。
- 不容易漏空格、漏引号、漏换行。
- 对多行内容特别友好，比如 HTML 模板、邮件正文、SQL 片段。

#### 3. 插值里可以放什么

- 变量
- 运算表达式
- 函数调用
- 三元表达式

例如：

```js
const isVip = true;
const price = 100;

const result = `final price: ${isVip ? price * 0.8 : price}`;
```

#### 4. 多行字符串的价值

```js
const html = `
  <div class="card">
    <h1>${name}</h1>
    <p>age: ${age}</p>
  </div>
`;
```

- 这在前端模板拼接、服务端生成邮件内容、日志格式化里都很常见。

#### 5. 标签模板字符串

- 模板字符串不只是“更方便的字符串拼接”，它还支持标签模板。
- 可以在模板被真正拼接前，对字符串片段和变量做自定义处理。

例如：

```js
function highlight(strings, ...values) {
  return strings.reduce((acc, str, index) => {
    const value = values[index] ?? "";
    return acc + str + `<b>${value}</b>`;
  }, "");
}

const result = highlight`Hello ${name}, age is ${age}`;
```

- 这类能力常用于：
  - 国际化处理
  - SQL / HTML 转义
  - CSS-in-JS

#### 6. 面试里可以补一句

- 模板字符串本质上提升的是“表达能力”和“可维护性”，不仅仅是少写几个加号。

#### 4. 解构赋值

- 支持数组、对象快速取值。
- 常用于函数参数、React props、接口返回值处理。

#### 5. 对象和类

- 对象字面量增强：属性简写、方法简写、计算属性名。
- `class`、`extends`、`super` 让基于原型的继承写法更清晰。

这里建议分成“对象字面量增强”和“类语法糖”两部分讲。

#### 1. 对象字面量增强

##### 属性简写

当变量名和属性名相同时，可以直接简写。

```js
const name = "Tom";
const age = 20;

const user = {
  name,
  age,
};
```

等价于：

```js
const user = {
  name: name,
  age: age,
};
```

##### 方法简写

```js
const user = {
  name: "Tom",
  sayHi() {
    console.log(`Hi, I am ${this.name}`);
  },
};
```

等价于：

```js
const user = {
  name: "Tom",
  sayHi: function () {
    console.log(`Hi, I am ${this.name}`);
  },
};
```

##### 计算属性名

- 属性名可以动态计算。

```js
const key = "score";

const student = {
  name: "Jerry",
  [key]: 95,
  [`level_${1 + 1}`]: "A",
};
```

这在以下场景很常用：

- 根据接口字段动态组装对象
- 做字典映射表
- 根据枚举值生成配置对象

#### 2. `class` 是什么

- `class` 本质上不是新的继承机制，而是对原型链继承的语法糖。
- JavaScript 底层依然是原型和原型链。

例如：

```js
class Person {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    console.log(`Hi, I am ${this.name}`);
  }
}

const p = new Person("Tom");
p.sayHi();
```

#### 3. `extends` 和 `super`

- `extends` 用于继承父类。
- `super()` 用于在子类构造器中调用父类构造函数。
- 在子类构造器里，如果用了 `this`，必须先调用 `super()`。

```js
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} makes a sound`);
  }
}

class Dog extends Animal {
  constructor(name, color) {
    super(name);
    this.color = color;
  }

  speak() {
    super.speak();
    console.log(`${this.name} barks`);
  }
}
```

#### 4. 为什么说它“让继承写法更清晰”

- 在 ES5 中，要手动处理构造函数继承、原型链继承、`constructor` 指回、方法挂载等细节。
- ES6 `class` 把这些表达成更接近传统面向对象语言的写法，可读性更强。

#### 5. 但要强调一个本质

- `class` 只是语法层更清晰，不代表 JavaScript 变成了“基于类的语言”。
- 本质仍然是基于原型的对象系统。

#### 6. 模块化

- `import` / `export` 提供原生模块系统。
- 支持静态分析，利于 Tree Shaking。

#### 7. 异步与集合

- `Promise` 标准化异步编排。
- `Set`、`Map`、`WeakSet`、`WeakMap` 提供更合理的数据结构。

这一块面试非常常问，建议从“为什么不直接用对象和数组”切入。

#### 1. `Set`

- `Set` 是值的集合，特点是成员唯一，不允许重复。

```js
const set = new Set([1, 2, 2, 3]);
console.log([...set]); // [1, 2, 3]
```

常见用途：

- 数组去重
- 快速判断某个值是否存在
- 维护选中项、黑名单、权限集合

常用方法：

- `add`
- `delete`
- `has`
- `clear`

数组去重：

```js
const arr = [1, 2, 2, 3];
const unique = [...new Set(arr)];
```

#### 2. `Map`

- `Map` 是键值对集合。
- 和普通对象不同，`Map` 的键可以是任意类型，不仅仅是字符串或 `Symbol`。

```js
const map = new Map();
const objKey = { id: 1 };

map.set(objKey, "value");
map.set("name", "Tom");

console.log(map.get(objKey)); // value
```

相比普通对象的优势：

- 键类型更灵活。
- 原生支持遍历。
- `size` 直接拿长度。
- 语义更明确，就是“映射表”。

常见场景：

- 缓存
- 配置映射
- 节点和元数据关联
- 需要对象作为 key 的场景

#### 3. 为什么说 `Set` / `Map` 更合理

- 当需求是“唯一集合”时，用 `Set` 比数组更语义化。
- 当需求是“键值映射”时，用 `Map` 比对象更合适，特别是 key 不固定或不是字符串时。
- 也就是说，它们解决的是“数据结构表达不准确”的问题。

#### 4. `WeakSet` 和 `WeakMap`

- 它们的 key 或成员只能是对象，而且是弱引用。
- 弱引用的意思是：如果外部没有别的强引用指向这个对象，垃圾回收可以直接回收它。

##### `WeakMap`

```js
let obj = { id: 1 };
const weakMap = new WeakMap();

weakMap.set(obj, { cache: true });

obj = null;
// 原对象如果没有其他引用，就可以被 GC 回收
```

特点：

- key 只能是对象。
- 不可遍历。
- 没有 `size`。
- 适合存放对象的附加信息，又不想影响垃圾回收。

常见场景：

- DOM 节点元数据缓存
- 私有数据关联
- 避免内存泄漏的缓存结构

##### `WeakSet`

- 成员只能是对象。
- 也是弱引用，不可遍历。

常见场景：

- 标记某个对象“是否处理过”
- 记录访问过的对象，避免重复处理

#### 5. 为什么弱引用结构不能遍历

- 因为其中对象可能随时被垃圾回收。
- 一旦允许遍历，遍历结果在语义上就不稳定。
- 所以规范直接限制它们不可枚举、不可遍历。

#### 6. 面试中常见对比

`Set` vs 数组：

- 数组适合有序列表。
- `Set` 适合去重和存在性判断。

`Map` vs 对象：

- 对象更适合固定结构数据。
- `Map` 更适合动态映射关系。

`Map` vs `WeakMap`：

- `Map` 可遍历、可统计大小，适合普通映射。
- `WeakMap` 不可遍历，但更适合和对象生命周期绑定的缓存。

#### 8. 迭代和生成器

- `for...of`、迭代器协议、`Generator`。
- 后续 `async/await` 的理解也建立在这些机制之上。

#### 9. 常用补充

- `Symbol`
- `Proxy` / `Reflect`
- `Object.assign`
- `Array.from`
- `find` / `findIndex`

#### 深入讲 `Proxy` / `Reflect`

#### 1. `Proxy` 是什么

- `Proxy` 可以理解为“对象的代理层”。
- 你可以在读取、设置、删除、函数调用、`in` 判断等操作发生时进行拦截。
- 它常用于响应式系统、数据校验、访问控制、日志埋点。

基础示例：

```js
const target = {
  name: "Tom",
};

const proxy = new Proxy(target, {
  get(target, key) {
    console.log("get:", key);
    return target[key];
  },
  set(target, key, value) {
    console.log("set:", key, value);
    target[key] = value;
    return true;
  },
});

proxy.name; // get: name
proxy.age = 20; // set: age 20
```

#### 2. 它能拦截什么

常见 trap 包括：

- `get`
- `set`
- `has`
- `deleteProperty`
- `ownKeys`
- `apply`
- `construct`

例如校验赋值：

```js
const user = new Proxy(
  {},
  {
    set(target, key, value) {
      if (key === "age" && typeof value !== "number") {
        throw new TypeError("age must be a number");
      }
      target[key] = value;
      return true;
    },
  }
);
```

#### 3. `Reflect` 是什么

- `Reflect` 是一组和对象操作对应的标准 API。
- 它不是“反射黑科技”，更像“把原本分散的对象内部操作，统一成函数形式暴露出来”。

例如：

```js
Reflect.get(obj, "name");
Reflect.set(obj, "age", 20);
Reflect.has(obj, "name");
Reflect.deleteProperty(obj, "name");
```

#### 4. 为什么 `Proxy` 常和 `Reflect` 一起使用

- 在 `Proxy` 的 trap 中，我们经常想“拦截一下，但默认行为还是照常执行”。
- 这时最规范的方式就是调用 `Reflect`。

例如：

```js
const proxy = new Proxy(target, {
  get(target, key, receiver) {
    console.log("reading", key);
    return Reflect.get(target, key, receiver);
  },
  set(target, key, value, receiver) {
    console.log("writing", key, value);
    return Reflect.set(target, key, value, receiver);
  },
});
```

这样做的好处：

- 不容易破坏原始语义。
- 能正确处理 `this`、原型链等细节。
- 返回值更规范，比如 `set` 要返回布尔值。

#### 5. Vue 3 为什么用 `Proxy`

- Vue 2 基于 `Object.defineProperty` 做响应式。
- Vue 3 改用 `Proxy`，因为它能直接代理整个对象，而不是逐个属性劫持。
- 对新增属性、删除属性、数组索引修改、`length` 变化等场景支持更自然。

#### 6. `Proxy` 的局限

- 代理的是对象，不是原对象本身。
- 对同一个目标对象的身份判断要注意代理对象和原对象不同。
- 兼容性虽然现代环境基本没问题，但老旧环境不如 `defineProperty`。

这里“代理的是对象，不是原对象本身”的意思是：

- `Proxy` 不会把原对象改造成“自带拦截能力的对象”。
- 它会返回一个新的代理对象，你后续必须通过这个代理对象去读写，拦截才会生效。
- 如果你绕过代理，直接操作原对象，那么 `get` / `set` 这些 trap 就不会触发。

还要注意身份问题：

```js
console.log(proxy === target); // false
```

- 说明代理对象和原对象不是同一个引用。
- 所以如果业务里做对象身份比较、`Map` key、缓存命中判断，要明确自己存的是代理对象还是原对象。

再举一个常见误区：

```js
const list = [];
const proxyList = new Proxy(list, {});

console.log(proxyList === list); // false
```

- 它们内容可以同步，因为底层操作的还是同一个目标对象。
- 但“引用身份”不是同一个。

一句话总结：

- `Proxy` 是“在原对象外面包了一层壳”。
- 你通过这层壳访问，才能被拦截。
- 直接碰原对象，就绕过了代理。

这句话在实际开发里主要影响 3 件事：

#### 1. 身份判断可能出错

```js
const target = { id: 1 };
const proxy = new Proxy(target, {});

console.log(proxy === target); // false
```

- 所以如果你在做引用比较、去重、缓存命中、`Map` / `Set` 存取时，要统一使用同一种对象身份。
- 否则你以为是“同一个对象”，代码判断出来却不是。

#### 2. 绕过代理会让监听失效

```js
const target = { name: "Tom" };
const proxy = new Proxy(target, {
  set(target, key, value, receiver) {
    console.log("trigger set");
    return Reflect.set(target, key, value, receiver);
  },
});

target.name = "Jerry"; // 不触发
proxy.name = "Lucy"; // 触发
```

- 所以响应式系统、状态管理系统如果基于 `Proxy`，通常都会要求你“始终操作代理对象”，否则依赖收集和更新通知会失效。

#### 3. 不是所有嵌套对象都会天然被代理

```js
const target = {
  info: { age: 18 },
};

const proxy = new Proxy(target, {});
```

- `proxy` 代理的是外层对象。
- 至于 `info` 这个内部对象是否也具备拦截能力，要看框架或你的实现有没有继续对它做代理。
- 这也是为什么像 Vue 3 这类响应式系统，通常会在读取嵌套对象时继续返回新的代理对象，而不是只代理最外层一次。

面试里可以这样总结这句话：

- `Proxy` 不是修改原对象本体，而是返回一个“代理引用”。
- 因此要注意两件事：一是 `proxy !== target`，二是直接操作 `target` 会绕过拦截。

#### 7. 面试里一句话总结

- `Proxy` 负责“拦截”。
- `Reflect` 负责“规范地执行默认对象操作”。
- 二者组合后，非常适合实现响应式、校验、访问控制和元编程能力。

### 讲讲类组件

#### 1. 类组件是什么

- React 早期通过 `class extends React.Component` 定义组件。
- 组件需要实现 `render()`，返回 JSX。
- 状态存放在 `this.state` 中，通过 `this.setState()` 更新。

#### 2. 核心能力

- 管理本地状态。
- 通过生命周期方法参与挂载、更新、卸载过程。
- 可以做副作用、订阅、定时器、性能优化等工作。

#### 3. 常见生命周期

挂载阶段：

- `constructor`
- `render`
- `componentDidMount`

更新阶段：

- `shouldComponentUpdate`
- `render`
- `componentDidUpdate`

卸载阶段：

- `componentWillUnmount`

错误处理：

- `componentDidCatch`
- `getDerivedStateFromError`

#### 4. `setState` 特点

- `setState` 是异步调度，不一定立刻更新。
- 多次更新可能被批处理。
- 依赖旧状态时建议使用函数式写法：

```js
this.setState((prev) => ({ count: prev.count + 1 }));
```

#### 5. 类组件的不足

- 逻辑复用不方便，常依赖 HOC、render props。
- `this` 绑定容易出错。
- 生命周期分散，相关逻辑往往被拆到多个方法中。
- 相比 Hooks，状态逻辑复用和组合能力偏弱。

#### 6. 现在怎么看类组件

- 现代 React 主流是函数组件 + Hooks。
- 但类组件仍然需要会，尤其在老项目维护、错误边界、面试原理题中很常见。

### 4. 给有序数组 `arr = [-4, -1, 0, 3, 5]`，返回平方后的排序

#### 直接写法

```js
const result = arr.map((item) => item * item).sort((a, b) => a - b);
```

结果：

```js
[0, 1, 9, 16, 25]
```

#### 时间复杂度

- `map` 是 `O(n)`
- `sort` 通常按比较排序理解为 `O(n log n)`
- 总体时间复杂度：`O(n log n)`

#### 优化思路

- 原数组已经有序，但平方后“负数绝对值大的项”可能跑到右边。
- 最大平方值一定出现在数组两端。
- 用双指针分别指向左右两端，比较绝对值，把更大的平方从结果数组尾部开始填充。

#### 优化写法

```js
function sortedSquares(arr) {
  const result = new Array(arr.length);
  let left = 0;
  let right = arr.length - 1;
  let index = arr.length - 1;

  while (left <= right) {
    const leftSquare = arr[left] * arr[left];
    const rightSquare = arr[right] * arr[right];

    if (leftSquare > rightSquare) {
      result[index] = leftSquare;
      left++;
    } else {
      result[index] = rightSquare;
      right--;
    }

    index--;
  }

  return result;
}
```

#### 优化后复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

#### 面试加分点

- 如果允许修改原数组，也很难把空间降到真正的 `O(1)` 且保持实现清晰。
- 这题的重点不是平方，而是“利用原数组有序”降低排序成本。
