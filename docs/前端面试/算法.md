# 算法训练题纲

## 使用方式

算法训练的目标不是“题量好看”，而是稳定通过面试。你做每道题都要能口头输出下面 5 件事：

- 题型归类
- 核心思路
- 时间复杂度和空间复杂度
- 关键边界条件
- 如果写错，最可能错在哪

算法面试里最常见的问题不是不会，而是：

- 看到题不知道先判题型
- 只会说思路，不会说复杂度
- 代码写出来了，但边界一问就崩
- 有优化解，但讲不清为什么更优

所以这份文档重点不是再塞代码，而是帮你建立“讲题框架”。

## 一、总目标

- 总题量目标：`150~220`
- 最近 `30` 题正确率：`80%+`
- 中等题平均完成时间：`20~30` 分钟
- 每周至少 `1` 次限时模拟

除此之外，更关键的是建立下面三个能力：

- 看到题先归类
- 写完后能主动说复杂度和边界
- 被追问时知道如何从朴素解升级到优化解

## 二、统一讲题模板

建议所有算法题都按下面顺序回答：

1. 先判题型
2. 说最直接的思路和为什么可行
3. 如果有更优解，说明为什么优化
4. 写代码时同步解释关键变量
5. 写完立刻说时间复杂度、空间复杂度
6. 最后补边界和常见坑

例如面试官给你一道字符串题，不要一上来就写，先判断：

- 是哈希表计数吗
- 是双指针 / 滑动窗口吗
- 是回文中心扩散吗
- 是 DP 吗

题型判对了，后面才容易稳。

## 三、题型识别速记

### 什么时候优先想到哈希表

- 需要快速查找某个值是否出现过
- 需要记录频次、位置、映射关系
- 题目强调线性时间

### 什么时候优先想到双指针 / 滑动窗口

- 数组或字符串连续区间问题
- 需要维护一个满足条件的子数组 / 子串
- 题目包含“最长”“最短”“不重复”“覆盖”这类关键词

### 什么时候优先想到链表指针

- 题目围绕删除、反转、合并、找中点、找环
- 不方便随机访问，只能顺着改指针

### 什么时候优先想到栈 / 单调栈

- 匹配括号
- 找下一个更大 / 更小元素
- 求区间贡献或柱状图面积

### 什么时候优先想到树递归

- 当前答案依赖左右子树信息
- 题目天然有“自顶向下”或“自底向上”结构

### 什么时候优先想到 BFS

- 求最短步数
- 按层遍历
- 多源扩散

### 什么时候优先想到回溯

- 要求列出所有方案
- 需要枚举排列、组合、子集
- 题目里有“所有可能”这种信号词

### 什么时候优先想到 DP

- 问最值、方案数、可行性
- 当前状态能由更小子问题推导
- 存在明显重复子问题

## 四、数组与双指针

1. 什么场景应该优先想到双指针？
2. 左右指针和快慢指针分别适合什么题型？
3. 有序数组两数之和怎么做？
4. 移除重复元素怎么做？
5. 盛最多水的容器怎么做？为什么贪心移动短板？
6. 三数之和怎么做？去重时要注意什么？
7. 接雨水有哪些做法？各自复杂度如何？
8. 滑动窗口和双指针的关系是什么？

### 这一类题的题目本质

- 双指针：本质是利用顺序结构减少重复扫描
- 滑动窗口：本质是维护一个动态区间
- 容器 / 接雨水：本质是用局部信息逐步收缩搜索空间

### 复杂度要会讲什么

- 多数双指针题目标是 `O(n)`
- 三数之和通常是排序后 `O(n^2)`
- 接雨水要能区分暴力、前后缀数组、双指针、单调栈几种解法的复杂度

### 常见易错点

- 指针移动条件写反
- 去重时漏掉重复分支
- 窗口扩张后忘了收缩
- 先更新答案还是先移动指针的顺序出错

### 推荐讲解顺序

1. 先判断是否能用顺序性降复杂度
2. 再解释两个指针各代表什么
3. 说明移动规则为什么正确
4. 写代码
5. 最后补边界和复杂度

## 五、字符串

9. 最长不重复子串怎么做？
10. 最小覆盖子串为什么是经典滑动窗口题？
11. 判断回文串有哪些常见方式？
12. 最长回文子串有哪些思路？
13. 字符串匹配问题中哈希能解决什么？
14. 字符串分割、解析、模拟题常见坑有哪些？
15. 字符串题什么时候可以转成数组或哈希表问题？

### 这一类题的题目本质

- 最长不重复子串 / 最小覆盖子串：窗口维护
- 回文问题：中心扩散、双指针或 DP
- 字符统计类：哈希计数
- 解析模拟类：状态管理和细节处理

### 复杂度要会讲什么

- 最长不重复子串通常追求 `O(n)`
- 最小覆盖子串也是经典 `O(n)` 窗口
- 最长回文子串要知道中心扩散 `O(n^2)`，Manacher 了解即可，不是主流手写要求

### 常见易错点

- 窗口中计数加减混乱
- 认为“包含”成立后就可以立刻结束
- 回文边界没处理好
- 字符串和字符数组转换时索引错位

### 推荐讲解顺序

1. 先判断是不是连续区间问题
2. 说明窗口维护条件或回文判断条件
3. 明确何时扩、何时缩
4. 写代码
5. 最后补边界字符串和空串情况

## 六、链表

16. 反转链表怎么做？递归和迭代有什么区别？
17. 判断链表是否有环怎么做？
18. 环的入口节点怎么找？
19. 合并两个有序链表怎么做？
20. 删除链表倒数第 `N` 个节点怎么做？
21. 两两交换链表节点怎么做？
22. K 个一组反转链表为什么经常被追问？
23. 链表题最常见边界错误是什么？

### 这一类题的题目本质

- 反转 / 合并 / 删除：指针重连
- 环检测：快慢指针
- K 组反转：局部反转 + 分组边界控制

### 复杂度要会讲什么

- 大多数链表题目标是 `O(n)`
- 额外空间一般追求 `O(1)`，如果用递归要说明递归栈空间

### 常见易错点

- 没保存下一个节点就改指针
- 头节点变化后没有 dummy 节点兜底
- 快慢指针初始位置不清楚
- 删除倒数第 `N` 个节点时边界为头节点删除

### 推荐讲解顺序

1. 先画出指针关系
2. 说明每个指针的职责
3. 再开始写代码
4. 写完重点验空链表、单节点、删头节点

## 七、栈与队列

24. 有效括号怎么做？
25. 最小栈怎么设计？
26. 用栈实现队列、用队列实现栈怎么做？
27. 单调栈适合解决哪些问题？
28. 每日温度怎么做？为什么用单调栈？
29. 柱状图最大矩形怎么做？
30. 滑动窗口最大值有哪些做法？

### 这一类题的题目本质

- 括号匹配：最近未匹配结构
- 最小栈：主栈 + 辅助信息同步维护
- 单调栈：维护“候选答案”并快速淘汰无用元素
- 滑动窗口最大值：双端队列维护单调性

### 复杂度要会讲什么

- 有效括号通常是 `O(n)`
- 最小栈所有操作目标是 `O(1)`
- 每日温度、滑动窗口最大值、柱状图最大矩形都要能解释为什么能做到线性复杂度

### 常见易错点

- 只会背模板，不知道单调栈维护的是什么不变量
- 弹栈条件写错
- 队列题把“值”和“下标”混用
- 栈空判断漏掉

### 推荐讲解顺序

1. 先说为什么普通遍历不够高效
2. 再定义栈 / 队列里存什么
3. 说明入栈出栈条件
4. 写代码
5. 最后证明复杂度

## 八、哈希表

31. 两数之和为什么是哈希表入门题？
32. 字母异位词分组怎么做？
33. 最长连续序列怎么做？为什么能做到线性复杂度？
34. LRU 缓存怎么设计？
35. 什么时候该用哈希表换时间？
36. 哈希冲突和复杂度退化在面试中该怎么讲？

### 这一类题的题目本质

- 两数之和：查找补数
- 异位词分组：构造统一 key
- 最长连续序列：利用集合快速判存在
- LRU：哈希查找 + 顺序维护

### 复杂度要会讲什么

- 这类题通常目标是均摊 `O(1)` 查询
- 要知道哈希表的最坏复杂度不一定是 `O(1)`，但面试里通常按均摊分析

### 常见易错点

- 先插入再查找导致同一元素被重复使用
- key 设计不稳定
- 忽略重复元素
- 把 LRU 说成纯哈希表就能做

### 推荐讲解顺序

1. 先说明为什么要“用空间换时间”
2. 定义哈希表里存什么
3. 写主逻辑
4. 最后补 key 设计和复杂度分析

## 九、二叉树与递归

37. 二叉树前中后序遍历怎么写？
38. 递归遍历和迭代遍历的区别是什么？
39. 二叉树层序遍历怎么做？
40. 最大深度、最小深度如何求？
41. 判断是否是对称二叉树怎么做？
42. 判断是否是平衡二叉树怎么做？
43. 最近公共祖先怎么做？
44. 二叉树序列化与反序列化怎么做？
45. 路径总和、最大路径和为什么常被当成递归代表题？

### 这一类题的题目本质

- 树递归：当前节点答案依赖子树结果
- 层序遍历：BFS 按层扩展
- 最近公共祖先：在树结构里向上汇总信息
- 序列化：结构编码与还原

### 复杂度要会讲什么

- 多数树题时间复杂度是 `O(n)`
- 递归写法的空间复杂度要考虑树高 `h`
- 层序遍历的额外空间通常和某层最大宽度有关

### 常见易错点

- 递归出口写不清
- 最大深度和最小深度条件混淆
- 最近公共祖先返回条件不完整
- 序列化只会写遍历，不会讲空节点保留原因

### 推荐讲解顺序

1. 先判断是 DFS 还是 BFS
2. 如果是 DFS，说明函数返回值含义
3. 如果是 BFS，说明队列里放什么
4. 写代码
5. 最后补树为空、单节点、极端不平衡树的情况

## 十、二分查找

46. 二分查找的本质是什么？

51. 二分答案类问题是什么？

### 这一类题的题目本质

- 普通二分：利用单调性缩小区间
- 查找边界：寻找第一个满足条件的位置
- 二分答案：对“答案空间”做判定

### 复杂度要会讲什么

- 普通二分是 `O(log n)`
- 二分答案通常是 `O(log range * checkCost)`

### 常见易错点

- 区间定义不统一，闭区间和半开区间混用
- `mid` 更新后死循环
- 找左边界和右边界时返回值处理错误
- 明明没有单调性还强行二分

### 推荐讲解顺序

1. 先证明单调性或可判定性
2. 统一区间写法
3. 说明收缩规则
4. 写代码
5. 用小样例手推边界

## 十一、回溯

53. 全排列怎么做？
54. 子集问题怎么做？

58. 回溯中的剪枝思路怎么设计？
59. 为什么很多回溯题代码模板相似？

### 这一类题的题目本质

- 回溯就是在搜索树上试探、撤销、继续试探
- 不同题的差别通常在状态记录、选择列表和剪枝条件

### 复杂度要会讲什么

- 回溯题往往是指数级或阶乘级复杂度
- 面试里重点是能否用剪枝降低实际搜索量，而不是把复杂度说得很好看

### 常见易错点

- 忘了撤销现场
- 组合和排列的去重逻辑混淆
- 剪枝条件放错位置
- 结果集里直接推入同一个引用

### 推荐讲解顺序

1. 先定义路径、选择列表、结束条件
2. 再说剪枝规则
3. 写代码
4. 最后讲复杂度和为什么模板能复用

### 这一类题的题目本质

- 岛屿数量：连通块统计
- 腐烂的橘子：多源 BFS 分层扩散
- 课程表：有向图是否有环 / 拓扑排序
- 并查集：动态维护连通性

### 复杂度要会讲什么

- 常规图遍历通常是 `O(V + E)`
- BFS 空间复杂度通常和队列大小有关

- 把树题的思路直接套到图上，导致重复访问
- 拓扑排序不清楚入度的含义
- 多源 BFS 起点初始化不完整

### 推荐讲解顺序

1. 先把题转成图模型
2. 说明节点、边、访问状态的定义
3. 再决定 BFS / DFS / 拓扑 / 并查集
4. 写代码
5. 最后补复杂度和边界图情况

## 十三、动态规划

67. 动态规划的本质是什么？
68. 什么特征提示你应该考虑 DP？
69. 爬楼梯怎么做？它体现了什么 DP 思维？
70. 打家劫舍怎么做？
71. 最长递增子序列怎么做？
72. 最长公共子序列怎么做？
73. 零钱兑换怎么做？
74. 编辑距离怎么做？
75. 背包问题为什么是 DP 核心题型？
76. DP 题你应该如何定义状态、转移方程、初始化？

### 这一类题的题目本质

- DP 是把大问题拆成有重叠的子问题，并把中间结果存下来复用
- 最核心的不是背模板，而是会定义状态和转移

### 复杂度要会讲什么

- 要明确状态数和每个状态的转移成本
- 例如二维 DP 常见是 `O(mn)`
- 如果能做滚动数组优化，要说明空间为什么能压缩

### 常见易错点

- 状态定义不清楚
- 初始化漏掉第一行、第一列或 base case
- 转移方程写对了，但循环顺序错了
- 只会背爬楼梯，不会迁移到更复杂题

### 推荐讲解顺序

1. 先说明为什么暴力会重复计算
2. 定义状态
3. 写转移方程
4. 说明初始化
5. 再写代码和复杂度

## 十四、贪心

77. 贪心的核心前提是什么？
78. 跳跃游戏怎么做？
79. 分发饼干、无重叠区间、买卖股票这些题为什么适合贪心？
80. 贪心和动态规划如何区分？
81. 如何证明一个贪心策略是正确的？

### 这一类题的题目本质

- 每一步做局部最优，并保证能导向全局最优
- 难点通常不在代码，而在证明策略正确

### 复杂度要会讲什么

- 贪心题代码往往不复杂，但要讲是否需要排序
- 如果先排序，复杂度通常由排序主导

### 常见易错点

- 凭感觉贪，不会证明
- 明明需要 DP，却硬套贪心
- 只会写结论，不会解释为什么当前最优不会伤害全局最优

### 推荐讲解顺序

1. 先说局部最优策略
2. 再解释为什么这个策略成立
3. 写代码
4. 最后补复杂度和反例思路

## 十五、堆与优先队列

82. 堆适合解决什么问题？
83. 数组中第 K 个最大元素怎么做？
84. 前 K 个高频元素怎么做？
85. 合并 K 个有序链表怎么做？
86. 为什么堆常用于 Top K 问题？

### 这一类题的题目本质

- 堆用于动态维护一组元素中的最值或前 K 名
- 比起每次全量排序，堆更适合边遍历边维护候选集

### 复杂度要会讲什么

- 要会说清堆大小是 `k` 还是 `n`
- Top K 常见复杂度是 `O(n log k)`
- 合并 K 个有序链表常见是 `O(n log k)`

### 常见易错点

- 小顶堆和大顶堆方向用反
- 忽略堆大小控制
- 只会说“用堆更快”，说不清为什么是 `log k`

### 推荐讲解顺序

1. 先说为什么不直接排序
2. 再定义堆里存什么
3. 说明维护策略
4. 写代码
5. 最后补复杂度

## 十六、设计题与综合题

87. LRU 缓存怎么设计并手写？
88. LFU 缓存和 LRU 的差异是什么？
89. 手写发布订阅模式时，你如何设计数据结构？
90. 手写 Promise 调度器、任务池时，算法思路是什么？
91. 你如何在面试里快速拆解一道综合设计题？

### 这一类题的题目本质

- 设计题不是纯算法，也不是纯 API 模拟，而是“数据结构 + 操作复杂度 + 边界约束”的组合题
- 面试官更看重你能否先明确目标复杂度，再选合适结构

### 推荐讲解顺序

1. 先确认操作集合
2. 明确复杂度目标
3. 选择数据结构
4. 解释核心不变量
5. 再写关键操作

- 过一周再做仍能做出来
