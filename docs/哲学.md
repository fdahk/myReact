# **学习知识/解决问题的方法论**
- 学习的本质是学一系列的名词或概念背后所代表的不可再分或最通俗的基本事物、以及名词之间的相互作用。
- 信息需要在实践中验证才能确保正确
- 核心循环
  - 澄清：明确相关信息/概念：明确目标、成功标准、约束、可用资源、时间上限。
  - 建模：明确当前状态：列“已知/未知/假设”，画系统要素与因果（输入→过程→输出→反馈）。
  - 分解：分解问题/目标：以可交付物为导向，拆成最小可验证子问题（MVP/行走骨架）。
  - 实验：验证是否取得正确贡献：设可证伪假设与衡量指标，设计最便宜的试验（一次只变一个变量）。
  - 复盘：更新状态，归纳收获：结果→差距→5 Whys 根因→更新模型与清单。
  - 系统化：把结论沉淀成模板/清单/范式，纳入间隔复习与检索。

- 学习与解决的最高优先事项
  - 先抓杠杆点：顶层原理、关键术语、常见陷阱、常用工具。
  - 成果倒逼学习：以“能交付”为目标，而非“看懂为止”。
  - 双峰策略：80% 做通用核心，20% 做一次深潜

- 评估与决策的通用刻度
  - 指标：正确性/速度/成本/鲁棒性/可复用/可解释。
  - 门槛（复杂度）：可逆/不可逆→可逆则优先快速尝试，不可逆则慢想。
  - 评估期望值：价值×成功率−成本；优先高 EV 且快反馈的任务。

- 检索与记忆（让学到的留下来）
  - 检索练习：闭卷回忆、费曼解释、出题自测。
  - 结构化记录：事实笔记 vs 永久笔记（洞见/范式）；链接相似/相反/因果。
  - 间隔重复：对关键术语、易混概念、常错题做间隔复习。

- 创造性与迁移（知识/技能 复用）
  - 类比/反转：A 像什么？如果反过来会怎样？极端/边界会怎样？
  - TRIZ/SCAMPER：替换、组合、适配、修改、用途变更、消除、重排。
  - 抽象梯度：在“具体例子—抽象原理—新场景”之间往返迁移。

- 卡住时的六步自救
  - 重述问题（换表述/画图/列式），缩小到更小的可验证单位。
  - 换约束（放宽或固定一个维度），做极端与反例。
  - 对比“已知的相似问题”（迁移/复用）。
  - 时间箱 + 基线方案（先尝试有解，再优化）。
  - 橡皮鸭讲解/同伴反馈/提问模板（背景→尝试→卡点→期望）。
  - 休息与切换任务（缓冲一下，让后台加工）。

规划 =》 收集信息 =》 验证效果 + 收集新信息 =》 确定方案 =》 执行（循环）

# **了解新领域的优先信息清单**
- 第一层：本质问题（建立认知框架）
1. 本体论 - 这个领域"是什么"
研究/处理的对象是什么？ （物理学研究物质与能量，经济学研究资源配置）
核心概念/术语是什么？ （不懂术语 = 无法进入这个领域的对话）
边界在哪？ 什么属于这个领域？什么不属于？
2. 价值论 - "为什么重要"
解决什么问题/满足什么需求？
对谁有价值？ （从业者/用户/社会）
不存在这个领域会怎样？ （反向思考其价值）
3. 认识论 - "怎么判断对错"
这个领域的"真理标准"是什么？
科学领域：可证伪、可重复实验
艺术领域：审美共识、历史地位
商业领域：市场验证、财务指标
法律领域：判例、条文解释
什么是这个领域的"好"与"坏"？
4. 方法论 - "怎么做事"
核心方法/工具是什么？ （科学家用实验，律师用案例分析，设计师用原型）
基本工作流程是什么？
入门的最小实践是什么？ （相当于技术领域的Hello World）

- 第二层：结构问题（建立地图）
5. 历史与演化
这个领域是如何诞生的？ （解决什么历史问题）
关键转折点/里程碑是什么？ （范式转移）
当前处于什么阶段？ （新兴/成熟/衰退）
未来趋势是什么？
6. 内部结构
有哪些子领域/分支？
不同流派/学派的分歧是什么？ （几乎每个领域都有派系之争）
核心争议/未解决的问题是什么？
7. 外部关系
与哪些领域相关/交叉？
从哪些领域借鉴方法？
对哪些领域产生影响？

- 第三层：实用问题（快速上手）
8. 权威与资源
谁是公认的权威/大师？ （读谁的书、听谁的话）
经典著作/入门材料是什么？
去哪里获取可靠信息？ （期刊/社区/机构）
9. 常见陷阱
外行最容易犯的错误是什么？
常见的误解/迷思是什么？
"看起来对但其实错"的观点？
10. 实践路径
从零到入门的最短路径？
如何验证自己"学会了"？
找谁请教/获得反馈？
通用提问模板

- 进入任何新领域时，按顺序问这些问题：
1. 这个领域研究/处理什么？（对象）2. 为什么这个领域存在？解决什么问题？（价值）3. 这个领域的"行话"有哪些？（术语）4. 怎么判断这个领域里什么是对的、什么是好的？（标准）5. 这个领域的人日常怎么工作？（方法）6. 这个领域有哪些派系/分支？他们在争什么？（结构）7. 这个领域的历史转折点是什么？（演化）8. 该读谁的书？听谁的话？（权威）9. 外行最容易踩什么坑？（陷阱）10. 我怎么最快"入门"？（路径）
快速评估模板
领域名称：___核心研究对象：___解决的核心问题：___判断对错的标准：___核心方法/工具：___主要流派/分支：___入门经典著作：___公认权威人物：___常见误区：___我的学习目的：___投入产出比评估：___

- 核心原则
先建框架，再填细节 — 不要一开始就陷入细节，先把"地图"画出来
找到"判断标准" — 每个领域都有自己的"真理观"，不懂这个就无法独立判断
识别派系争议 — 任何领域都有争议，了解争议才能理解全貌
警惕"外行视角" — 搜索"X common misconceptions"避免踩坑
以输出倒逼输入 — 设定一个小目标（写一篇文章/做一个小项目/解决一个具体问题）
这个框架本质上是在问：本体（是什么）→ 价值（为什么）→ 认识（怎么判断）→ 方法（怎么做）→ 结构（怎么组织）→ 资源（去哪学）




# **针对代码开发领域的新技术学习，信息掌握的优先级**
1. Why - 存在的理由（最优先）
解决什么问题？ 这个技术/领域诞生是为了解决什么痛点？
为什么现有方案不够好？ 对比之前的解决方案，它的改进点在哪？
价值主张是什么？ 它承诺带来什么好处？
理解 Why 判断"是否值得投入"，避免追逐不必要的新技术

2. What - 核心概念
关键术语定义：这个领域的"行话"是什么？不懂术语就无法沟通
核心概念及其关系：画出概念之间的关系图（输入→处理→输出）
边界：它能做什么？不能做什么？（不能做什么往往更重要）

3. Where - 定位与对比
与现有方案的对比：这个技术在整个生态中的位置
适用场景 vs 不适用场景：什么时候该用？什么时候不该用？
优缺点权衡：没有银弹，了解代价是什么

4. How - 最小可用路径
Hello World：最小可运行示例，建立"能跑通"的信心
核心工作流：日常使用的主要流程是什么？
工具链/生态：配套工具、常用库、IDE支持

5. Who - 社区与生态
谁在维护？ 个人/公司/基金会？（影响长期可靠性）
社区活跃度：GitHub stars、issues处理速度、Stack Overflow问题数
行业采用：哪些知名公司/项目在用？

6. Trap - 常见陷阱
新手常犯的错误（搜索 "X common mistakes" / "X gotchas"）
性能/安全隐患
版本兼容问题

实操建议
优先级	行动	时间投入
P0	读官方 "Getting Started" / "Why X" 页面	30分钟
P0	跑通一个最小示例	1小时
P1	对比文章（X vs Y）	30分钟
P1	搜索 "X best practices" / "X common mistakes"	30分钟
P2	看一个完整项目的代码结构	1-2小时
P2	了解社区/生态状况	30分钟
快速评估模板
技术名称：___解决的核心问题：___适用场景：___不适用场景：___主要替代方案：___学习曲线（1-5）：___社区活跃度（1-5）：___是否值得深入：是/否/待定
> 核心原则：先建立"能用"的最小闭环，再逐步深入。不要一开始就追求"完全理解"。


# **如何高效沟通**
- 先对齐对齐概念理解，达成基本共识，否则后续所有沟通都是有偏差的
- 明确沟通目的，明确双方沟通希望达成的目的，否则沟通过程很难对齐思路

# **如何准确、高效地理解概念？**
- 概念（名词）为什么取这个名字？很多概念是自解释的，发明者必然需要一个名词来准确的描述、表达，这个名词本质就是对一系列事物高度精炼的准确概括
- 理解不同概念（名词）间的相互作用机制，一个领域由若干概念组成，概念间必然通过某种关系或机制才能共同发挥作用。其中关系或机制本身也是会被该领域专家精炼成名词
- 明确概念边界，不同概念间的边界、重合、冲突部分要明确，否则概念同时需要处理时会造成严重误差
