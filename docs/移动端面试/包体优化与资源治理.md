# 包体优化与资源治理

## 使用方式

包体题不要只回答“图片压缩、按需加载”。真正高分的回答会同时讲下载、安装、解压、启动、运行时内存和版本治理。

---

## 一、为什么移动端比 Web 更看重包体

1. 移动端安装包会直接影响下载转化率。
2. 包体越大，安装、解压、启动准备成本越高。
3. 低端机和弱网下，包体问题会被放大得更明显。
4. 包体通常还和内存占用、资源解码、更新速度相关。

---

## 二、包体问题通常来自哪里

### 1. 代码

- 无效依赖
- 重复依赖
- 调试代码和历史兼容代码未清理
- 首包塞入过多低频模块

### 2. 资源

- 图片尺寸过大
- 同一资源多份格式重复存在
- 字体文件太大
- 动画资源、视频资源缺乏治理

### 3. 平台能力

- 插件或 SDK 过多
- 引擎体积
- 多渠道多 ABI 资源冗余

---

## 三、包体优化怎么拆层回答

### 1. 首包控制

只把高频和首屏强依赖能力放进首包，其他模块动态加载或延后准备。

### 2. 资源治理

统一做：

- 图片尺寸规范
- 图片格式选择
- 字体和图标策略
- 资源去重

### 3. 依赖治理

定期审视：

- 依赖是否真的在用
- 能不能用更轻量方案
- 是否引入了一整个库只用了一个小功能

### 4. 发布策略

包体优化不能和发布割裂。比如资源热更新、离线包、分包、动态模块，都要和版本治理一起设计。

---

## 四、不同技术栈怎么看包体

### `React Native`

要看：

- JS bundle 大小
- 图片和字体资源
- 原生依赖和 SDK 数量
- Hermes 等运行时选择带来的影响

### `Flutter`

要看：

- 引擎基础体积
- 插件数量
- 资源目录管理
- 不同平台构建产物

### 小程序

要看：

- 首包大小
- 分包策略
- 公共资源复用

### `Hybrid`

要看：

- 容器本身大小
- H5 静态资源体积
- 离线包和缓存策略

---

## 五、资源治理怎么答

### 图片

重点讲：

- 展示尺寸和原图尺寸匹配
- 是否使用更合适格式
- 是否有缩略图策略
- 是否按场景延迟加载

### 字体和图标

重点讲：

- 是否真的需要整套字体
- 是否可以字体子集化
- 是否能用矢量图标替代大图

### 多媒体

重点讲：

- 音视频资源是否按需加载
- 预览资源和正式资源是否分层

---

## 六、为什么包体优化不能只靠一次性清理

因为它是一个持续治理问题：

- 业务会不断加功能
- 第三方 SDK 会持续接入
- 活动资源会反复堆积

所以高分回答通常会主动提到：

- 包体监控
- 资源规范
- 发版前检查项
- 依赖审计

---

## 七、典型场景题

1. 包体突然变大，你会先看什么。
2. 为什么首包已经不大了，启动还是慢。
3. 图很多的页面为什么会同时拖慢包体和性能。
4. 引入一个第三方 SDK 前要评估什么。
5. 分包做了以后为什么仍然不一定快。

---

## 八、回答模板

如果面试官问“你做过哪些包体优化”，建议按这个顺序说：

1. 先讲包体问题影响了什么。
2. 再按代码、资源、SDK、分包四层拆。
3. 再讲你做了哪些治理动作。
4. 最后讲结果和如何长期防止反弹。
