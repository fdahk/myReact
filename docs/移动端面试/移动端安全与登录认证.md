# 移动端安全与登录认证

## 使用方式

移动端安全题不要只背 `HTTPS`、`token`、`XSS`。真正高频的问题是：登录态怎么存、宿主和 H5 如何共享态、token 过期如何刷新、敏感能力怎么做权限和风控。

---

## 一、移动端安全和 Web 安全有什么不同

1. 移动端除了前端页面本身，还要考虑宿主、桥接和本地存储。
2. 登录态可能由原生账号系统管理，前端只是消费。
3. 很多能力会触达支付、定位、文件、相机等敏感系统权限。
4. 老版本长期在线，安全策略升级要考虑兼容。

---

## 二、登录态怎么答

高分回答建议覆盖完整链路：

1. 用户登录获取凭证。
2. 宿主或前端存储凭证。
3. 请求层统一携带。
4. 服务端校验。
5. 过期刷新。
6. 退出时统一清理。

同时要说明：前端负责的是状态维护和用户体验，真正的鉴权裁决始终在服务端。

---

## 三、token 存储为什么是高频题

因为移动端常见几种情况：

- 纯前端页面自己管理 token
- App 宿主统一管理 token，再透传给 H5 或跨端页面
- 多容器、多页面共享同一账号体系

回答时要体现：

1. 存储位置影响安全和可用性。
2. H5 与宿主共享态不能只靠页面缓存。
3. token 刷新策略必须统一。

---

## 四、权限和敏感能力怎么答

移动端高频敏感能力包括：

- 定位
- 相机
- 麦克风
- 相册
- 通知
- 文件读写

高分回答要体现：

1. 权限申请要按需，不要一股脑全要。
2. 被拒绝后要有产品兜底和引导。
3. 权限结果要和真实业务链路联动，不只是弹个系统框。

---

## 五、桥接安全为什么重要

因为 H5 或跨端页面一旦能调用原生能力，桥接协议本身就成了安全边界的一部分。

高分回答建议主动提：

- 只暴露必要能力
- 参数校验
- 来源校验
- 错误码和日志
- 高风险能力增加宿主侧二次确认

---

## 六、移动端安全常见场景

1. token 过期后多个请求同时刷新。
2. H5 页面和宿主登录态不同步。
3. 旧版本页面访问了新桥接能力。
4. 敏感能力被拒绝后页面逻辑崩掉。
5. 支付、分享、文件等高风险链路缺少状态校验。

---

## 七、典型追问

1. 为什么前端权限控制不是安全边界？
答：因为前端更多是体验控制，真正的数据和操作权限必须由服务端或宿主能力侧兜底。

2. 为什么移动端登录态题经常会追到宿主？
答：因为很多 App 的账号体系根本不在单个页面里，页面只是其中一个消费端。

3. 为什么 token 刷新一定要考虑并发？
答：因为多个请求同时过期会引发刷新风暴，处理不好就会放大失败面和状态错乱。

---

## 八、回答模板

如果面试官问“移动端登录态一般怎么设计”，建议按这个顺序说：

1. 先讲凭证来源和存储位置。
2. 再讲请求层如何统一携带和刷新。
3. 再讲 H5、跨端页面和宿主如何同步状态。
4. 最后讲敏感能力、权限和安全兜底。
