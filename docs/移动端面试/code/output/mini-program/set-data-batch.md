## 4. 小程序 `setData` 批量更新理解题

### 题目

```js
Page({
  data: {
    count: 0,
  },
  onLoad() {
    this.setData({ count: this.data.count + 1 });
    this.setData({ count: this.data.count + 1 });
    console.log(this.data.count);
  },
});
```

### 常见误判

很多人会以为这里最终是 `2`。

### 分析结论

这道题的关键不在死记某个固定输出，而在理解：

1. `setData` 不是普通对象赋值。
2. `this.data` 的读取时机和视图层同步时机不能简单等同。
3. 如果你连续基于旧值去计算新值，很容易出现更新结果和预期不一致。

### 面试里应该怎么回答

更稳妥的表达是：

- 小程序里连续 `setData` 时，不应该假设每次都已经完成了完整的视图同步。
- 这种写法本身就有可读性和正确性风险。
- 真正安全的做法是先在逻辑层算出最终值，再一次性 `setData`。

### 移动端场景映射

这类题对应的真实问题通常是：

1. 高频输入和频繁联动时，开发者误把 `setData` 当同步变量赋值用。
2. 连续基于旧 `data` 计算新状态，会放大逻辑层和渲染层通信成本，也容易引入错乱。
