# 公司相关

## 美国公司联合创始人协议版本说明

在准备美国公司语境下的 `Co-Founder Confirmation Agreement` 或类似联合创始人确认协议时，常见会出现三个版本思路：

- 通用美国公司版
- `Delaware C-Corp` 版
- `California-friendly` 版

这三个版本的核心区别，不在于“是不是联合创始人协议”，而在于它们分别针对不同的公司形态、州法环境和实际签约风险做了调整。

## 一、通用美国公司版

### 1. 定位

通用美国公司版是一个偏中性、可复用的基础模板，适合作为初稿或母版。它通常不预设公司一定注册在特拉华州，也不默认公司已经采用了非常标准的融资型股权架构。

### 2. 适用场景

适合以下情况：

- 公司刚成立，很多法律和股权安排还没有完全定型
- 还没有明确未来一定走美国 VC 融资路线
- 公司注册州、实际经营地、创始团队所在地还在调整
- 需要先快速形成一份结构完整、表述正式的联合创始人协议

### 3. 条款特点

这个版本通常会：

- 保留基础条款，例如 `co-founder status`、`equity`、`confidentiality`、`IP assignment`、`dispute resolution`
- 把很多关键字段留空，例如 `governing law`、`venue`、`arbitration`、`equity document type`
- 对股权安排只做确认性表述，不深入展开复杂的融资术语
- 对竞业限制、董事会批准、税务处理等问题写得相对克制

### 4. 优点

- 最容易快速落地
- 可作为多个州版本的基础底稿
- 对尚未完全规范化的初创团队更友好

### 5. 风险和局限

- 通用版虽然覆盖面广，但法律适配性较弱
- 如果公司未来融资，投资人律师通常会希望协议语言更贴近标准融资文件体系
- 如果实际经营高度依赖某一州，例如加州，通用版里部分限制性条款可能需要进一步调整

## 二、Delaware C-Corp 版

### 1. 定位

`Delaware C-Corp` 版是为美国科技创业公司最常见的融资架构准备的版本。美国很多准备融资的初创公司，尤其是面向机构投资人的公司，都会采用 `Delaware C-Corporation` 结构。

### 2. 为什么 Delaware 特别重要

特拉华州在美国公司法体系中非常特殊，原因包括：

- 公司法成熟，判例丰富
- 投资人、律师、创业者都非常熟悉
- 董事会、股东、优先股、融资轮文件等制度化程度高
- 美国 VC 融资文件通常天然更贴近 `Delaware C-Corp` 逻辑

因此，如果公司未来目标是：

- 融资
- 发行期权
- 采用标准员工股权激励计划
- 与美国投资人签署成套法律文件

那么 `Delaware C-Corp` 版通常更合适。

### 3. 条款特点

相较通用版，这个版本通常会更强调：

- `Board approval`
- `Equity Incentive Plan`
- `Restricted Stock`
- `Stock Option`
- `vesting`
- `repurchase rights`
- `83(b) election`
- 与公司章程、股东协议、董事会决议之间的衔接关系

也就是说，它不仅确认“你是联合创始人”，还会更自然地把这个身份放进一整套美国标准融资公司治理框架中。

### 4. 优点

- 更符合美国融资型创业公司的主流实践
- 更容易与后续融资、期权池、董事会治理文件衔接
- 对律师和投资人来说理解成本更低

### 5. 风险和局限

- 条款更专业，阅读门槛更高
- 如果公司其实不是 `Delaware C-Corp`，直接套用会出现不匹配
- 如果团队目前没有完整股权文件，这个版本可能显得“超前”

### 6. 典型适合对象

- 注册或准备重组为 `Delaware C-Corp` 的公司
- 希望未来接受美国机构投资的创业团队
- 已经在使用或准备使用标准股权激励计划的公司

## 三、California-friendly 版

### 1. 定位

`California-friendly` 版并不是一种公司类型，而是一种对加州法律环境更友好的协议写法。它的重点不是融资结构，而是**可执行性**和**合规风险控制**。

### 2. 为什么要单独区分 California

加州在劳动和限制性条款方面对个人保护更强，很多在其他州常见的写法，在加州可能存在较大执行风险，尤其包括：

- `non-compete`
- 某些过宽的 `non-solicitation`
- 对个人职业自由限制过强的条款

因此，如果公司、创始人、雇员、顾问、业务运营地或潜在争议处理地点与加州密切相关，就不能简单照搬其他州更“强硬”的限制性条款。

### 3. 条款特点

相较前两个版本，这个版本通常会：

- 弱化或删除 `non-compete`
- 更谨慎地表述 `non-solicitation`
- 把保护重点放在 `confidentiality` 和 `trade secret protection`
- 更明确说明某些限制仅在适用法律允许的范围内生效
- 避免写入明显容易被挑战的过度限制条款

### 4. 优点

- 更贴近加州实际法律环境
- 能降低协议局部条款被认定无效的风险
- 更适合创始团队、顾问、员工分布在加州的场景

### 5. 风险和局限

- 对公司一方的“控制力”保护会比其他州模板弱一些
- 如果公司不在加州、争议处理也不在加州，这个版本可能显得过于保守
- 如果需要融资，仍然需要与标准融资法律文件做进一步协调

### 6. 典型适合对象

- 公司在加州注册或主要经营
- 联合创始人本人常驻加州
- 团队关系更接近雇佣或顾问服务，需要考虑加州劳动法环境

## 四、三者最核心的区别

如果用一句话概括：

- 通用美国公司版：解决“先有一份完整、通用、能改的协议”
- `Delaware C-Corp` 版：解决“让协议适配标准美国融资公司结构”
- `California-friendly` 版：解决“让协议更贴近加州法律的可执行边界”

换句话说：

- 通用版是“基础模板”
- `Delaware` 版是“融资结构导向模板”
- `California-friendly` 版是“州法风险导向模板”

## 五、应该怎么选

### 1. 优先选通用版的情况

如果你们目前只是想先把联合创始人身份、贡献、股权比例和基础义务确认下来，但尚未进入标准融资体系，也还没有明确由哪个州法深度主导，那么先用通用版最稳妥。

### 2. 优先选 Delaware 版的情况

如果公司就是 `Delaware C-Corp`，或者未来明确要按照美国 VC 常见路径融资，那么更建议直接采用 `Delaware C-Corp` 版。这样后续无论是：

- 期权授予
- 股份归属
- 董事会审批
- 融资尽调

都会更加顺畅。

### 3. 优先选 California-friendly 版的情况

如果公司或联合创始人与加州联系很强，尤其是：

- 人在加州
- 公司主要在加州经营
- 合作关系和劳动关系容易受到加州法影响

那么应该优先考虑 `California-friendly` 版，至少要对限制性条款进行单独调整。

## 六、实务建议

在真实签署前，建议至少确认以下问题：

### 1. 公司到底是什么结构

需要先确认：

- 是 `C-Corp` 还是 `LLC`
- 注册州是哪里
- 实际经营地是哪里

这些信息会直接影响协议中的：

- `governing law`
- `venue`
- 股权术语
- 公司治理引用文件

### 2. 股权到底是什么性质

中国语境里常说“给了 0.2% 股权/期权”，但在美国文件里，这两者差异很大。必须明确是：

- 已发行普通股
- 限制性股票
- 股票期权
- `profits interest`
- 未来授予承诺

不同工具对应的法律后果、税务处理和离职处理都不同。

### 3. 知识产权条款不能只写“归公司”

美国合同通常会更强调：

- `work made for hire`
- assignment
- further assurances
- prior inventions carve-out

如果只写一句“知识产权归公司”，往往不够。

### 4. 竞业限制不要照搬中文模板

在美国特别是加州，竞业限制的可执行性远低于很多中文合同模板的默认认知。实务上更常见、更稳妥的做法是：

- 强化保密义务
- 强化商业秘密保护
- 精准定义公司机密信息
- 只在合法边界内设置限制性义务

## 七、结论

这三个版本并不是“内容多少”的区别，而是“法律适配方向”的区别。

- 如果要一个起步模板，用通用美国公司版
- 如果目标是标准美国融资路径，用 `Delaware C-Corp` 版
- 如果与加州联系密切，优先做 `California-friendly` 调整

最稳妥的做法通常是：

先确定公司注册州、公司类型和股权工具，再从通用版或 `Delaware` 版出发，根据是否涉及加州因素做定制化修改。
